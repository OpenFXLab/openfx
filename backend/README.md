# Backend

This directory will contain the OpenFXLab backend application.

**Status: Not yet implemented.** The backend is planned for Phase 1 (data foundation) and Phase 2 (public MVP) of the roadmap.

---

## Intended purpose

The backend serves two primary functions:

1. **Data ingestion** — Scheduled Python jobs that download, validate, normalize, and store positioning and macro data from official public sources.
2. **API service** — A FastAPI application that serves positioning and macro data to the web interface, API clients, and research tools.

---

## Proposed responsibilities

### Data ingestion layer

- Download CFTC Traders in Financial Futures (TFF) reports weekly
- Parse and normalize raw CFTC report files
- Map CFTC contract codes to currency pairs
- Apply currency direction normalization
- Ingest macro indicators from official sources (central bank rates, inflation, employment, growth)
- Ingest currency price data from an appropriately licensed source
- Validate all ingested data against defined schemas and quality rules
- Store raw source files without modification
- Store normalized, analytical data in PostgreSQL
- Alert on ingestion failures or stale data

### API service

- Serve current and historical positioning data by currency and participant group
- Serve macro indicator data
- Serve screener results based on query parameters
- Serve data download endpoints
- Document all endpoints via OpenAPI (auto-generated by FastAPI)
- Rate limit requests in later versions
